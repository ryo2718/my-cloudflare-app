# プリフロップトレーニング 用語定義

中級ポジション別モード(EP/LP/Blind)で導入した出題・採点の用語。
以降の開発はこの定義で扱う。各実装ファイルの先頭コメントからも本ドキュメントを参照する。

## スライダー形式問題 (slider question)

1 アクションの頻度を **0–100%・10% 刻みのスライダー**で回答する形式。
「この問題を飛ばす」ボタン、および端へ一発で寄せる **0% / 100% ボタン**あり。

**2 択ノードはスライダー形式に統一する**(`resolveSliderConversion`)。能動側のアクションを
100% 軸、降りる/受ける側を 0% 軸にする:

- コール / フォールド(相手オールイン = vs 5bet)→ **コール頻度**を軸(`sliderAction='call'`)
- レイズ / チェック(BB vs limp)→ **レイズ頻度**を軸(`sliderAction='raise'`)
- open / vs open(raise 頻度を問う既存スライダー)→ raise 軸(従来どおり)

正解頻度 = GTO の `strategy[sliderAction]`(別データは持たない)。
**3 択以上のノードは複数選択形式のまま**(下記)。

採点(`sliderScoring.ts`):

- 正解が **端**(0% または 100%):±0% → **2pt** / ±10% → **1pt** / それ以外 → **-1pt**
- 正解が **中間**(10–90%):±10% → **2pt** / ±20% → **1pt** / それ以外 → **-1pt**
- 「飛ばす」 → **0pt**(満点母数は固定。減点も加点もしない)
- 時間切れ → **-1pt**(既存ルール踏襲)

## 複数選択形式問題 (multi-select question)

**3 択以上**のノードのみ(4 択固定=オールイン/レイズ/コール/フォールド、SB open の
レイズ/リンプ/フォールド 等)。2 択ノードはスライダー形式(上記)。
チェックボックスでアクションを複数選択し、頻度ベースで採点
(`preflopIntermediate.ts` の `scoreAnswer`)。Blind では限定的な救済(下記「limp 配点緩和」)を追加。
配点緩和は 3 択以上の limp 絡みノードのみで効く(2 択スライダーには適用しない)。

Blind はノード別に利用可能アクションが異なるため、ボタン集合・ラベルをノードから出し分ける
(例: SB open は「リンプ」=`call`、vs open は「コール」=`call`、BB vs limp は「チェック」=`check`)。

## 境界出題ルール (boundary band rule)

レンジ表で「100% 参加(raise/call)ギリギリのハンド」から「100% fold」までの**境界帯**のみ出題する。

抽出アルゴリズム(`boundaryBand.ts` の `extractBoundaryBand`):

1. ハンドを**ファミリー**でグループ化する。
   - ペア(AA…22)= 1 ファミリー
   - スーテッド = 高位カード + `s`(例: `As` = AKs…A2s、`9s` = 98s…92s)
   - オフスート = 高位カード + `o`(例: `Ao` = AKo…A2o)
2. ファミリー内をキッカー(ペアはランク)の**強い順**に並べる。
3. **最弱の「≈100% 参加」ハンド**から、そこより弱い側で**最初の「≈0% 参加」ハンド**まで(両端含む連続帯)を境界帯とする。
   ≈0% に達しなければファミリー最弱まで。参加% = `100 - fold`。

実データでの検証例(指示書と一致):
Axs UTG = A3s〜A2s / Kxs UTG = K9s〜K4s / ペア HJ = 66〜22 / Axo HJ = ATo〜A8o。

## 全ハンド出題ルール (all-hand rule)

境界帯に限定せず**全ハンド**から出題する(EP の vs3bet / vs4bet で使用)。
ただしオフハンドが過剰にならないよう、**同じ 2 ランクのオフ/スーテッド(KJo と KJs 等)を 1 枠**として扱い、
抽選で枠が選ばれたらその場でどちらか一方をランダムに出す。ペアはそれぞれ 1 枠
(`boundaryBand.ts` の `collapseToSlots` / `pickFromSlot`)。

## ジャム受け出題上限 (jam-receive cap, 中級EP 専用)

中級EP(UTG/HJ)の1セッション(20問)で、**ジャム受け問題(`ep_vs_4bet` = vs 5bet,
コール/フォールドの2択スライダー)は最大3問**。空いた分は `open` / `vs 3bet` に
比率維持で補充し、合計20問・満点20pt・クリア基準(80%)は不変
(`MODE_RECIPES.ep` = open 8 / vs3bet 9 / vs4bet 3)。中級EP のみ。LP/Blind/総合/初級には適用しない。

## BB ライト3bet 下限保証 (light 3bet quota, 中級Blind 専用)

中級Blind の BB vs open(`bb_vs_open_other` 6題 / `bb_vs_open_sb` 3題, 計9題)で、
**最低3題はライト3bet候補ハンドから出題する**(下限保証 = クオータ。配分は
`bb_vs_open_other` 2題 + `bb_vs_open_sb` 1題、`LIGHT_3BET_QUOTA`)。残りは従来どおり
境界出題ルールから(コール/フォールド境界も残す)。

**ライト3bet候補**(`lightThreeBetCandidates`)= 次を全て満たすハンド:
1. 境界帯(`extractBoundaryBand`)に含まれる、2. そのノードで `0 < raise(=3bet) < 100`
(raise=100% の確定3betは除外)、3. UTG オープンレンジ(`utg.json`)で `raise < 100`
(UTG が100%openするハンドはライト3betでないため除外)。

題数配分(6/3)・全30問・満点30pt・クリア基準(80%=24pt)・出題形式(4択)・採点は不変。
候補プールが下限に満たない場合のみ通常出題にフォールバック(問題数維持)。中級Blind のみ。

## ハンド重複制約 (no duplicate hand, 全モード)

1セッション内で **同一シナリオに同じハンドを2回出さない**(`scenario:hand` 単位)。
異なるシナリオなら同じハンドが出てよい(例: open の 99 と vs3bet の 99 は両方可)。
出題プールのユニークハンド数がそのシナリオの問題数より少ない場合
(例 `lp_vs_4bet`: ユニーク4 < 6)のみ、末尾リトライで重複を許容して問題数を維持する。

## limp 配点緩和 (limp relaxation, Blind 専用)

通常採点が **-1pt** になるケースのうち、「**降りるべきでないハンドで、配分は外したが降りずに参加した**」
場合を **+1pt** に救済する。

発動条件(`preflopIntermediatePositional.ts` の `scoreBlindAnswer`):

- GTO が **raise + limp 主体**(raise + limp ≥ 70%)かつ **fold が僅少**
- かつユーザー選択が **raise / limp のみ**(limp = ノードに応じて `call` または `check`)で、**fold を含まない**
- → 通常の -1pt を **+1pt** に上書き

救済しないケース:
- fold を 1 つでも選んでいる(降りる判断が混ざる)
- **allin は参加扱いに含めない**(コミット度が異なる別判断)。allin が絡む場合は通常採点
