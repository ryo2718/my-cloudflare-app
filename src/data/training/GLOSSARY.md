# プリフロップトレーニング 用語定義

中級ポジション別モード(EP/LP/Blind)で導入した出題・採点の用語。
以降の開発はこの定義で扱う。各実装ファイルの先頭コメントからも本ドキュメントを参照する。

## スライダー形式問題 (slider question)

1 アクション(主に raise)の頻度を **0–100%・10% 刻みのスライダー**で回答する形式。
「この問題を飛ばす」ボタンあり。

採点(`sliderScoring.ts`):

- 正解が **端**(0% または 100%):±0% → **2pt** / ±10% → **1pt** / それ以外 → **-1pt**
- 正解が **中間**(10–90%):±10% → **2pt** / ±20% → **1pt** / それ以外 → **-1pt**
- 「飛ばす」 → **0pt**(満点母数は固定。減点も加点もしない)
- 時間切れ → **-1pt**(既存ルール踏襲)

## 複数選択形式問題 (multi-select question)

既存の中級総合と同じ形式。チェックボックスでアクションを複数選択し、頻度ベースで採点
(`preflopIntermediate.ts` の `scoreAnswer`)。Blind では限定的な救済(下記「limp 配点緩和」)を追加。

Blind はノード別に利用可能アクションが異なるため、ボタン集合・ラベルをノードから出し分ける
(例: SB open は「リンプ」=`call`、vs open は「コール」=`call`、BB vs limp は「チェック」=`check`)。

## 境界出題ルール (boundary band rule)

レンジ表で「100% 参加(raise/call)ギリギリのハンド」から「100% fold」までの**境界帯**のみ出題する。

抽出アルゴリズム(`boundaryBand.ts` の `extractBoundaryBand`):

1. ハンドを**ファミリー**でグループ化する。
   - ペア(AA…22)= 1 ファミリー
   - スーテッド = 高位カード + `s`(例: `As` = AKs…A2s、`9s` = 98s…92s)
   - オフスート = 高位カード + `o`(例: `Ao` = AKo…A2o)
2. ファミリー内をキッカー(ペアはランク)の**強い順**に並べる。
3. **最弱の「≈100% 参加」ハンド**から、そこより弱い側で**最初の「≈0% 参加」ハンド**まで(両端含む連続帯)を境界帯とする。
   ≈0% に達しなければファミリー最弱まで。参加% = `100 - fold`。

実データでの検証例(指示書と一致):
Axs UTG = A3s〜A2s / Kxs UTG = K9s〜K4s / ペア HJ = 66〜22 / Axo HJ = ATo〜A8o。

## 全ハンド出題ルール (all-hand rule)

境界帯に限定せず**全ハンド**から出題する(EP の vs3bet / vs4bet で使用)。
ただしオフハンドが過剰にならないよう、**同じ 2 ランクのオフ/スーテッド(KJo と KJs 等)を 1 枠**として扱い、
抽選で枠が選ばれたらその場でどちらか一方をランダムに出す。ペアはそれぞれ 1 枠
(`boundaryBand.ts` の `collapseToSlots` / `pickFromSlot`)。

## limp 配点緩和 (limp relaxation, Blind 専用)

通常採点が **-1pt** になるケースのうち、「**降りるべきでないハンドで、配分は外したが降りずに参加した**」
場合を **+1pt** に救済する。

発動条件(`preflopIntermediatePositional.ts` の `scoreBlindAnswer`):

- GTO が **raise + limp 主体**(raise + limp ≥ 70%)かつ **fold が僅少**
- かつユーザー選択が **raise / limp のみ**(limp = ノードに応じて `call` または `check`)で、**fold を含まない**
- → 通常の -1pt を **+1pt** に上書き

救済しないケース:
- fold を 1 つでも選んでいる(降りる判断が混ざる)
- **allin は参加扱いに含めない**(コミット度が異なる別判断)。allin が絡む場合は通常採点
