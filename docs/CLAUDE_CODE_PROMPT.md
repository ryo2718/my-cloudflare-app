## 初回プロンプト (これをClaude Codeに送信)

```
ポーカーのプリフロップGTO戦略表示Webアプリを作っています。
前任のClaude (claude.aiのチャット版) と要件・スキーマ・初期コードまで決めて、
ここから実装フェーズです。

【まずやってほしいこと】
1. README.md を読んで全体像を把握
2. SCHEMA.md でJSONフォーマットを確認
3. DECISIONS.md で設計判断の理由を確認
4. preflop-viewer.jsx と rfi_utg.json で雛形を確認
5. 上記を踏まえて、Phase 1 (Viteプロジェクト初期化〜コンポーネント分割) に着手

【重要なルール】
- スキーマ (SCHEMA.md) は契約。勝手に変更しない
- ソルバー本体はアプリに含めない (AGPL感染回避)
- 計算ロジックはこのリポジトリでは書かない (別リポジトリで管理予定)
- ハンドの粒度: プリフロップは169通り、フロップは1225組み合わせ

【マシン環境】
- Windows 11 Pro
- Node.js環境想定 (npm利用可)
- VSCode系エディタ想定

【進め方の希望】
- 不明点はボタン形式で質問してください (選択肢2-4個)
- ポーカー用語 (RFI, 3bet, GTO, SRP等) は通じます
- 大きな判断は私に確認してから進めてください
- 小さな実装判断はある程度任せます

まずREADME.mdから読み始めてください。
```

## Tips: Claude Codeでの作業を効率化するコツ

### セッション開始時
- 必ず引き継ぎ資料を最初に読ませる
- いきなりコードを書かせない (文脈を理解させる)

### 作業中
- 重要な判断は必ず確認 (ファイル削除、スキーマ変更、ライブラリ選定)
- 「コミットしていい？」と聞いてから作業を区切る

### つまづいた時
- このプロジェクトは前任Claudeとの相談履歴があるので、似た問題は前任に聞き返してもいい
- 「DECISIONS.md に書いてある？」と聞くと判断の根拠が見つかる

### スキーマ変更が必要になった時
- 必ず `schema_version` をアップデート
- 既存データとの互換性を考慮
- `SCHEMA.md` も同時に更新

## トラブルシューティング

### Q: Claude Codeが計算ロジックを書こうとしている
A: 「このリポジトリは表示専用。計算は別リポジトリ予定」と伝える。
   DECISIONS.md の「計算ロジックと表示ロジックの分離」を参照。

### Q: Claude Codeがソルバーをアプリに組み込もうとしている
A: 「AGPL感染するのでNG」と伝える。
   DECISIONS.md の「AGPL-3.0の扱い」を参照。

### Q: スキーマを勝手に変えようとしている
A: 「スキーマは契約、勝手に変えない」と伝える。
   変更が必要なら理由を明記して `schema_version` をバンプ。
